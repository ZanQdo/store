File Wrangler - User Manual

1. Introduction

Welcome to File Wrangler! This Blender addon is a powerful batch processing tool designed to automate repetitive tasks across multiple .blend files. Whether you need to clean up entire project directories, change file settings, or run custom scripts, File Wrangler saves you hours of manual work by handling it all in one go.

The core philosophy is to set up a series of actions, apply filters to target specific files, and execute the process safely and efficiently.

2. Installation

  1. Download the file_wrangler.zip file.

  2. In Blender, go to Edit > Preferences > Add-ons.

  3. Click the Install... button at the top-right.

  4. Navigate to and select the downloaded .zip file.

  5. Enable the addon by ticking the checkbox next to its name.

The addon panel will appear in the 3D View's Sidebar (press the N key) under the "File Wrangler" tab.

3. The File Wrangler Interface

You can find the File Wrangler panel in the 3D Viewport. Press the N key to open the sidebar, and look for a tab named "File Wrangler".

The panel is organized into three main sections: Setup & I/O, Filename Filters, and Actions.

4. Step-by-Step Guide

Section 1: Setup & I/O

This section defines what files to process and where to save them.

  • Source Directory: This is the most important setting. Select the main folder containing the .blend files you want to process.

  • Output Directory (Optional but Recommended):

    • If you set an Output Directory: File Wrangler will copy the original folder structure from your Source Directory into the Output Directory and save all modified files there. This is the safest, non-destructive way to work.

    • If you leave this blank: File Wrangler will overwrite your original files in the Source Directory. Use this with extreme caution.

  • Search Recursively: When enabled, File Wrangler will search for .blend files inside all subfolders of your Source Directory. When disabled, it will only look in the top level.

  • Write Log File: If enabled, a detailed text file named file_wrangler_log.txt will be created. The log records every action taken, every file processed, and any errors that occurred.

Section 2: Filename Filters

Filters allow you to precisely target files within your Source Directory. This is useful for processing only characters, environments, or files from a specific shot.

  • Include files containing: Enable this to process only the files whose names contain the specified keywords. You can list multiple keywords separated by commas (e.g., character, prop, car).

  • Exclude files containing: Enable this to skip any files whose names contain the specified keywords. This is useful for avoiding test files or assets you don't want to change (e.g., test, old, broken).

Section 3: Actions

Here you choose what operations to perform on the targeted .blend files. You can enable multiple actions at once.

  • Delete Backup Files (.blend1): This action scans the Source Directory (recursively, if enabled) and deletes all Blender backup files (e.g., my_file.blend1, my_file.blend2). This is a great way to clean up project folders. This action runs independently of the main file processing loop.

  • Purge Unused Data: For each .blend file, this will perform a recursive "Orphans Purge," which removes unused data-blocks like materials, textures, and node groups that are not assigned to any object. This helps reduce file size and clutter.

  • Find Missing Files: This action attempts to automatically locate and relink missing external files (like image textures).

    • Search Path: You can specify a folder where File Wrangler should look for these missing files. If you leave it blank, it will default to searching within your main Source Directory.

  • Run Custom Script: A powerful feature for advanced users. This allows you to run your own Python script on every .blend file.

    • Script Source: You can choose where the script comes from:

      • External File: Lets you select a .py script from your computer.

      • Internal Text: Lets you select a script from Blender's internal Text Editor. This is useful for scripts that are part of your project.

    • Inside the script, you have access to the bpy and os modules to perform almost any operation.

  • File Paths: This action changes how paths to external files are stored within the .blend file.

    • Don't Change: Leaves paths as they are.

    • Relative: Makes all paths relative to the location of the .blend file. Good for portable projects.

    • Absolute: Makes all paths absolute (e.g., C:\Users\You\Desktop\...). Good for studios with fixed file structures.

  • File Compression: This changes the compression setting of the .blend file itself.

    • Don't Change: Leaves the compression setting as it is.

    • Compress: Saves the file with compression enabled, resulting in a smaller file size.

    • Decompress: Saves the file without compression. Files will be larger but may open slightly faster.

5. Executing the Process

Once you have your settings, filters, and actions configured, you have two options at the bottom of the panel:

  • Dry Run: This is a safe preview. It will scan your directories and apply your filters, then print a list of all the files that would be processed to the Blender System Console (and the log file, if enabled). It will not make any changes to your files. Always use this first to ensure you're targeting the correct files.

  • Start Batch Process: This begins the operation. A progress bar will appear at the top of the File Wrangler panel showing the current status. While running, a Cancel Process button will be available to safely stop the process after it finishes the current file.

⚠️ Important Warnings

  • ALWAYS BACK UP YOUR DATA before running a batch process, especially if you are not using a separate Output Directory.

  • Actions like "Delete Backup Files" and overwriting files are permanent. Double-check your settings.

  • Using a separate Output Directory is the recommended workflow for safety.

6. Preferences

You can customize the name of the UI tab in Blender's preferences.

  1. Go to Edit > Preferences > Add-ons.

  2. Find "File Wrangler" and expand its settings by clicking the small arrow.

  3. You can change the text in the UI Tab Name field. The change will take effect immediately.
