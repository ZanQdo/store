File Wrangler User Manual

Batch Processing Automation for Blender

01 Introduction

Welcome to File Wrangler! This powerful pipeline tool automates repetitive tasks across multiple .blend files. Whether you need to clean up entire project directories, change file settings, run custom Python scripts, or unify workspace layouts, File Wrangler saves you hours of manual work by handling it all in one go.

The core philosophy is simple: set your target files, configure your actions, and let the batch processor safely execute them in the background.

02 Installation

File Wrangler is packaged as a standard Blender 4.2+ Extension.

  1. Download the file_wrangler.zip file.
  2. In Blender, go to Edit > Preferences > Get Extensions.
  3. Click the drop-down arrow at the top right and select Install from Disk...
  4. Select the downloaded .zip file.
  5. Ensure the extension is enabled by ticking the checkbox.

03 Setup & Inputs

Press N in the 3D Viewport to open the Sidebar and locate the File Wrangler tab.

3.1 Input Modes

Choose how to target your files. File Wrangler offers two distinct workflows:

Directory Search

Target an entire folder. Supports recursive searching (scanning all subdirectories) and filtering using inclusive/exclusive filename keywords.

Custom File List

Curate a precise batch. Manually add specific .blend files to a list, bypassing entire folders and filters completely.

3.2 Output & Logging

  • Output Directory (Recommended):
    • If set: Copies your exact folder structure and saves the modified files here. This is non-destructive and safe.
    • If blank: Overwrites the original files in place. Use with caution.
  • Write Log File: Creates a detailed file_wrangler_log.txt in the output directory, recording every modified file, error, or custom script printout.

04 Actions Toolkit

Select one or more actions to perform sequentially on every targeted file.

Overwrite UI

Forces all processed files to adopt your current window layout and workspace setup upon saving.

Run Custom Script

Execute a Python script on every file. Load from an external .py file or select an internal text block.

Delete Backup Files

Scans and permanently deletes auto-saved backups (e.g., .blend1) to free up hard drive space.

Purge Unused Data

Performs a recursive "Orphans Purge", cleaning out unassigned materials, meshes, node groups, and textures.

Find Missing Files

Automatically attempts to locate and relink broken image textures or external assets within a specified directory.

Paths & Compression

Convert all asset links to Relative or Absolute paths. Force files to save as Compressed (smaller) or Decompressed (faster loading).

05 Execution Modes

  • Dry Run: Always use this first! It prints a list of affected files to the System Console/Log without making any actual changes.
  • Start Batch Process: Runs the sequence interactively, allowing you to see progress and cancel at any time.
  • Run in Background: Spawns a separate, headless Blender instance to do the heavy lifting. This keeps your main UI completely responsive and protects your current session if a faulty file crashes the background worker.
⚠️ Important Warnings
  • ALWAYS BACK UP YOUR DATA before running an overwrite operation.
  • Setting an Output Directory is the safest way to batch process.
  • The "Delete Backup Files" operation bypasses the trash and is permanent.

06 Preferences

Customize where File Wrangler appears in the UI:

  1. Go to Edit > Preferences > Add-ons.
  2. Locate the "File Wrangler" preferences.
  3. Change the UI Tab Name field to move the tool to a different Sidebar category (e.g., "Pipeline").
