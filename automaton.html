Precision Scoping

Scripts only run exactly where they are allowed to.

⊞

Directory Scope

Use wildcards to restrict scripts to specific project folders. Tell a script to only execute if the currently opened `.blend` file lives inside the //assets/ directory.

⋈

File Filters

Include or exclude specific file names. Ensure your automated lighting setup script only triggers on files matching shot_*.blend and skips work-in-progress files.

Production Safeguards

Keep automation helpful, not annoying.

⏱ Smart Cooldowns
Prevent heavy scripts from running constantly. Set an hourly cooldown, or restrict a script to run exactly Once Per Session until Blender is restarted.
◈ Local Overrides
Sometimes a specific shot requires breaking the rules. Artists can easily disable global rules locally from the N-Panel without affecting the rest of the project.
◎ Dry Run Scanning
Unsure if your wildcards are correct? Use the Dry Run feature to rapidly scan your entire project hierarchy and print a list of every `.blend` file that matches your criteria.

Built for Teams

Automaton saves its configurations directly inside your scripts directory as automaton_config.json. Sync this file via any distributed file system or network drive alongside your python scripts, and your entire team's rule-set updates instantly across the studio.
