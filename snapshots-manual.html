Snapshots - User Manual

1. Introduction

Welcome to Snapshots, a professional version-control workflow for Blender. Snapshots is designed to liberate your creative process by removing the fear of commitment. By creating non-linear "checkpoints" in your work, you can explore radical creative tangents, A/B test lighting setups, and prototype complex procedural systems, all while knowing your primary vision is safe and restorable with surgical precision.

2. Installation

  1. Download the snapshots.zip file.

  2. In Blender, go to Edit > Preferences > Add-ons.

  3. Click the Install... button at the top-right.

  4. Select the downloaded .zip file.

  5. Enable the addon by ticking the checkbox next to Snapshots.

3. Configuration & Pipeline

Access these settings in Edit > Preferences > Add-ons > Snapshots.

3.1 Global Snapshots Directory

By default, Snapshots are saved in a dedicated folder relative to your project.

  • Studio/Pipeline Use: If you are working on a high-speed network or have limited local storage space, set a Global Snapshots Directory to centralize all backups to a secondary drive or server.

3.2 UI Customization

Change the UI Tab Name to move the Snapshots panel to a specific location in your N-panel sidebar (e.g., "Pipeline" or "Exploration").

4. The Snapshots Sidebar

Press N in the 3D Viewport and select the Snapshots tab.

4.1 Creating a Snapshot

Click Create Snapshot.

  • The First Save: If your file hasn't been saved yet, the button will read Create Snapshot & Save. This initializes the snapshot system for your file and creates your first snapshot.

  • Descriptions: Use the dialog to add a short description (e.g., "Radical Silhouette Exploration").

4.2 Managing Your History

Each snapshot in the list is a collapsible container:

  • Annotations (Notes): Click the snapshot to expand it. You can type detailed notes or feedback directly into the text field to keep track of your creative process and specific ideas.

  • Actions (Cog Icon): * Rename: Update the snapshot description.

    • Delete: Permanently remove the snapshot from your project history.

5. Surgical Restoration

Snapshots allows you to mix and match different versions of your project. You don't always have to revert the whole file; you can pull just the ideas you need.

5.1 Right-Click Context Menus (Property Level)

This is the most powerful way to use Snapshots.

  1. Right-click any field (Roughness, Focal Length, Location, etc.).

  2. Select Restore Property from Snapshot.

  3. Pick a snapshot from the list. Only that specific value will revert.

5.2 Component Restoration (Data-Block Level)

You can restore entire data-blocks by right-clicking their names:

  • Materials: Right-click the material name in the Shader Editor or Properties panel to restore the entire node tree.

  • Modifiers & Constraints: Right-click the name of a modifier or constraint in the stack to revert its entire configuration.

  • Actions: Restore specific animation takes by right-clicking the Action name in the Action Editor.

5.3 Bulk Restoration (Viewport Menu)

Select multiple objects in the 3D Viewport and Right-click > Restore from Snapshot.

  • Restore Data: Reverts underlying meshes/curves/lights while keeping current positions.

  • Restore Object Properties: Reverts transforms (Location, Rotation, Scale).

  • Restore Materials: Reverts all material slot assignments for the selection.

6. Recovery & Scene Tools

6.1 Recover Deleted Data

If you set aside an idea earlier and want it back:

  1. Expand a snapshot in the sidebar.

  2. Click Recover Deleted Data.

  3. The dialog lists every object or data-block (World, Action, Palette, etc.) present in that snapshot but missing from your current scene.

  4. Select the items and click OK. Snapshots will bring them back instantly, even placing objects back into their original Collections.

6.2 Scene-Level Restoration

Inside the expanded snapshot panel, use the quick-action buttons to restore:

  • Scene: Render settings, units, and scene-wide configuration.

  • World: Environment lighting and world shader trees.

  • View Layer: Pass settings and collection visibility.

7. Technical Compatibility

Snapshots supports a massive range of Blender data, including:

  • Geometry: Meshes, Curves, Lattices, Text, and Armatures.

  • Grease Pencil: Full support for both Legacy and Grease Pencil v3.

  • Nodes: Geometry Node trees, Shader trees, and Compositor setups.

  • Metadata: Color Palettes, Freestyle Line Styles, and Texture data.
