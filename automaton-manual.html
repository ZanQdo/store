Automaton User Manual

Pipeline Automation for Blender

01 Introduction

Automaton is a powerful pipeline automation add-on for Blender that allows you to execute Python scripts automatically based on specific application events (like saving, rendering, or changing frames).

Whether you need to enforce naming conventions before saving, automatically bake textures, or set up render outputs when a file is loaded, Automaton allows you to define flexible Rules to "Automate anything."

02 Core Concepts

Automaton operates on three basic concepts:

Projects

Configurations that point to a specific working directory on your drive and a sub-folder containing your Python scripts.

Targets

The specific Python script (or directory of scripts) you want to execute.

Rules

The configuration that dictates when and where a Target should be executed.

03 Getting Started

3.1 Installation

  1. In Blender, go to Edit > Preferences > Add-ons.
  2. Click Install... and select the downloaded Automaton .zip or manifest file.
  3. Enable the checkbox next to Pipeline: Automaton.

3.2 Setting Up Your First Project

To begin automating, you must create a Project in the Add-on Preferences:

  • Open Automaton's preferences and click Add New Project.
  • Project Root: Pick the main, absolute directory of your current project on your hard drive.
  • Scripts Sub-path: Pick the relative folder inside your project where your automation .py scripts will live (defaults to automaton).
Shared Pipeline Configuration

Automaton saves a shared automaton_config.json file in this directory so your entire team can share the same automation rules!

Kitsu Users: Automaton automatically detects Kitsu project structures and will set the default scripts path to svn/tools/automaton if applicable.

⚠️ Security Recommendation

Because Automaton executes Python files directly, any script placed in the shared automaton folder will run arbitrary code on your team's machines. For studio environments, it is highly recommended to configure network or version control permissions so that the scripts folder is read-only for the majority of the team. Only trusted pipeline TDs or administrators should have write access to this folder and its configurations.

04 Creating Rules

Once your project is set up, you can add Rules in the preferences. A rule connects a script to a Blender event.

  • Click the + (Add) button under the Rules List.
  • Select your Target (a specific .py file or a whole folder of scripts).
  • Select your Trigger (the Blender event that will fire the script).

Available Triggers

Automaton hooks into Blender's native handlers. Major trigger categories include:

  • File Operations: Load (Pre/Post), Save (Pre/Post), Blend Import (Pre/Post).
  • Rendering & Compositing: Render (Init, Pre, Post, Write, Complete, Cancel), Composite (Pre, Post, Cancel).
  • Animation & Updates: Frame Change (Pre/Post), Animation Playback (Pre/Post), Depsgraph Update (Pre/Post).
  • Other Tools: Object Bake, Undo/Redo operations.
  • None (Manual Only): Use this if you only want the script to appear as a button in the UI, but never run automatically.

Execution Controls

You can prevent scripts from running too often using execution controls:

  • Once Per Session: If enabled, the rule will only execute the very first time the trigger happens. It will not run again until Blender is completely restarted.
  • Cooldown (Hours): Sets a minimum time limit between executions. For example, a cooldown of 2.0 means the script will only run once every 2 hours, regardless of how many times the trigger event occurs.

05 Filtering and Scoping (Where scripts run)

You likely don't want every script to run on every single .blend file. Automaton provides powerful path and filename scoping to strictly control execution.

Directory Scope (Include / Exclude)

Restricts the rule based on the folder the current .blend file is located in.

Wildcard Syntax:
  • * Matches any character or directory. (e.g., */characters/*)
  • ? Matches any single character. (e.g., shot_0?/)
  • // Starts the match strictly from the Project Root you defined. (e.g., //assets/)
Auto-Formatting:
  • Typing chars automatically becomes *chars*.
  • Typing //assets automatically becomes //assets/*.

File Filters (Include / Exclude)

Restricts the rule based on the name of the current .blend file.

Syntax:
  • * Matches any characters (e.g., *_rig.blend).
  • ? Matches any single character.
Auto-Formatting:

Typing rig automatically becomes *rig*.

Tip:

You can specify multiple paths or file names by separating them with commas (e.g., *_rig.blend, *_model.blend).

Dry Run Target List

Not sure if your wildcards are correct? In the rule details, click Scan Project for Matches. Automaton will quickly scan your project directory and print a list of every .blend file that matches your current filters into the System Console.

06 The N-Panel Interface (Manual Control)

Automaton provides a user interface in the 3D Viewport and the Video Sequencer (VSE). Open the side panel (shortcut N) and look for the Automaton tab.

6.1 Running Scripts Manually

  • The panel lists all valid scripts for the currently open .blend file (based on your Scopes and Filters).
  • Click the large Play button next to a script or directory name to manually execute it immediately, bypassing any triggers or cooldowns.

6.2 Local Overrides (Toggling Rules)

  • Underneath each script, you will see a checklist of its active automated rules.
  • Unchecking the box next to a rule creates a Local Override.
  • This completely disables that automation rule only for the current scene in the current blend file. This is saved within your .blend file, allowing artists to opt-out of specific automations when necessary without breaking the shared project pipeline.

07 Advanced Preferences

Under Edit > Preferences > Add-ons > Automaton, you can adjust global behaviors:

  • Tab Name: Change the name of the N-Panel tab (defaults to "Automaton"). Useful if you want to consolidate multiple add-ons into one tab.
  • Visibility: Toggle whether the Automaton panel appears in the 3D Viewport, the Sequence Editor, or both.
